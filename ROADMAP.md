# Roadmap

Planned phases for the HyperLocal delivery platform. Timelines are indicative and will be refined as implementation begins.

## Phase 0 — Foundation (Current)

- [x] Repository structure and documentation
- [ ] Backend project scaffold (runtime, lint, test tooling)
- [ ] Database schema design and migration strategy
- [ ] CI/CD pipeline and environment configuration
- [ ] Shared design system and API contract templates

## Phase 1 — Core Platform

- [ ] User authentication and authorization (customers, merchants, riders, admins)
- [ ] Merchant onboarding and store management
- [ ] Product/catalog management
- [ ] Service area and geofencing configuration
- [ ] Admin dashboard (basic operations)

## Phase 2 — Order & Fulfillment

- [ ] Shopping cart and checkout flow
- [ ] Order lifecycle (placed → confirmed → prepared → picked up → delivered)
- [ ] Rider assignment and dispatch logic
- [ ] Real-time order and location tracking
- [ ] Notifications (push, SMS, email)

## Phase 3 — Payments & Operations

- [ ] Payment gateway integration
- [ ] Refunds and dispute handling
- [ ] Pricing, fees, and commission rules
- [ ] Reporting and analytics
- [ ] Support tools and audit logs

## Phase 4 — Client Applications

- [ ] Customer web/mobile app
- [ ] Merchant partner app
- [ ] Rider/delivery partner app
- [ ] Operations admin panel

## Phase 5 — Scale & Polish

- [ ] Performance optimization and caching
- [ ] Multi-zone / multi-city support
- [ ] Promotions, loyalty, and referrals
- [ ] Localization and accessibility
- [ ] Production hardening, monitoring, and on-call runbooks

## Out of Scope (Initial Release)

- Cross-border shipping
- Marketplace seller auctions
- In-app voice/video support
